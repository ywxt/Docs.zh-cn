### <a name="how-the-membership-framework-handles-invalid-login-attempts"></a><span data-ttu-id="9b97e-150">成员资格框架如何处理无效的登录尝试次数</span><span class="sxs-lookup"><span data-stu-id="9b97e-150">How the Membership Framework Handles Invalid Login Attempts</span></span>

<span data-ttu-id="9b97e-151">当在访问者达到登录页，并提交其凭据时，其浏览器发出 HTTP 请求到登录页。</span><span class="sxs-lookup"><span data-stu-id="9b97e-151">When a visitor reaches the login page and submits their credentials, their browser makes an HTTP request to the login page.</span></span> <span data-ttu-id="9b97e-152">如果凭据有效，HTTP 响应将包括在 cookie 中的身份验证票证。</span><span class="sxs-lookup"><span data-stu-id="9b97e-152">If the credentials are valid, the HTTP response includes the authentication ticket in a cookie.</span></span> <span data-ttu-id="9b97e-153">因此，黑客试图进入你的站点无法创建彻底将 HTTP 请求发送到包含有效的用户名和密码猜测的登录页的程序。</span><span class="sxs-lookup"><span data-stu-id="9b97e-153">Therefore, a hacker attempting to break into your site could create a program that exhaustively sends HTTP requests to the login page with a valid username and a guess at the password.</span></span> <span data-ttu-id="9b97e-154">如果密码猜测值正确无误，请登录页将返回身份验证票证 cookie，此时该程序知道它有偶然发现有效的用户名/密码对。</span><span class="sxs-lookup"><span data-stu-id="9b97e-154">If the password guess is correct, the login page will return the authentication ticket cookie, at which point the program knows it has stumbled upon a valid username/password pair.</span></span> <span data-ttu-id="9b97e-155">通过暴力破解此类程序可能能够发现用户的密码，尤其是在弱密码时。</span><span class="sxs-lookup"><span data-stu-id="9b97e-155">Through brute force, such a program might be able to stumble upon a user's password, especially if the password is weak.</span></span>

<span data-ttu-id="9b97e-156">若要防止此类暴力破解攻击，成员资格框架锁定用户是否存在一定数量的一段时间内的失败登录尝试。</span><span class="sxs-lookup"><span data-stu-id="9b97e-156">To prevent such brute force attacks, the Membership framework locks out a user if there are a certain number of unsuccessful login attempts within a certain period of time.</span></span> <span data-ttu-id="9b97e-157">确切的参数是可通过以下两个成员资格提供程序配置设置配置：</span><span class="sxs-lookup"><span data-stu-id="9b97e-157">The exact parameters are configurable through the following two Membership provider configuration settings:</span></span>

- <span data-ttu-id="9b97e-158">`maxInvalidPasswordAttempts` -指定多少次无效密码尝试在该帐户锁定前的时间段内只允许用户。默认值为 5。</span><span class="sxs-lookup"><span data-stu-id="9b97e-158">`maxInvalidPasswordAttempts` - specifies how many invalid password attempts are allowed for the user within the time period before the account is locked out. The default value is 5.</span></span>
