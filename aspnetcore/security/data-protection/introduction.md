## <a name="problem-statement"></a><span data-ttu-id="d7486-109">问题陈述</span><span class="sxs-lookup"><span data-stu-id="d7486-109">Problem statement</span></span>

<span data-ttu-id="d7486-117">持久化的状态由服务器信任的因为我们预计这种状态可能包含特定于操作环境的信息。</span><span class="sxs-lookup"><span data-stu-id="d7486-117">Since the persisted state is trusted by the server, we anticipate that this state might contain information that's specific to the operating environment.</span></span> <span data-ttu-id="d7486-118">这可能是在文件路径、 权限、 句柄或其他间接引用的窗体或其他一些特定于服务器的数据。</span><span class="sxs-lookup"><span data-stu-id="d7486-118">This could be in the form of a file path, a permission, a handle or other indirect reference, or some other piece of server-specific data.</span></span> <span data-ttu-id="d7486-119">此类信息通常不应公开到不受信任的客户端。</span><span class="sxs-lookup"><span data-stu-id="d7486-119">Such information should generally not be disclosed to an untrusted client.</span></span> <span data-ttu-id="d7486-120">因此第二个要求： 保密性。</span><span class="sxs-lookup"><span data-stu-id="d7486-120">Thus the second requirement: confidentiality.</span></span>

<span data-ttu-id="d7486-121">最后，现代应用程序的组件化，因为我们已了解是各个组件将想要充分利用此系统而不考虑其他组件的系统中。</span><span class="sxs-lookup"><span data-stu-id="d7486-121">Finally, since modern applications are componentized, what we've seen is that individual components will want to take advantage of this system without regard to other components in the system.</span></span> <span data-ttu-id="d7486-122">例如，如果持有者令牌组件使用的此堆栈，它应运行不受干扰地从一种反 CSRF 机制，也可能使用相同的堆栈。</span><span class="sxs-lookup"><span data-stu-id="d7486-122">For instance, if a bearer token component is using this stack, it should operate without interference from an anti-CSRF mechanism that might also be using the same stack.</span></span> <span data-ttu-id="d7486-123">因此最后一项要求： 隔离。</span><span class="sxs-lookup"><span data-stu-id="d7486-123">Thus the final requirement: isolation.</span></span>

<span data-ttu-id="d7486-124">我们可以进一步约束以缩小我们的要求的范围。</span><span class="sxs-lookup"><span data-stu-id="d7486-124">We can provide further constraints in order to narrow the scope of our requirements.</span></span> <span data-ttu-id="d7486-125">我们假定加密系统内的所有服务都是同等信任和数据无需生成或在我们的直接控制服务外部使用。</span><span class="sxs-lookup"><span data-stu-id="d7486-125">We assume that all services operating within the cryptosystem are equally trusted and that the data doesn't need to be generated or consumed outside of the services under our direct control.</span></span> <span data-ttu-id="d7486-126">此外，我们需要的操作是尽可能快，因为 web 服务的每个请求可能会经过加密系统的一个或多个时间。</span><span class="sxs-lookup"><span data-stu-id="d7486-126">Furthermore, we require that operations are as fast as possible since each request to the web service might go through the cryptosystem one or more times.</span></span> <span data-ttu-id="d7486-127">这使得对称加密非常适合用于我们的方案，和我们可以折扣此类具有所需的时间之前的非对称加密。</span><span class="sxs-lookup"><span data-stu-id="d7486-127">This makes symmetric cryptography ideal for our scenario, and we can discount asymmetric cryptography until such a time that it's needed.</span></span>

## <a name="design-philosophy"></a><span data-ttu-id="d7486-128">设计理念</span><span class="sxs-lookup"><span data-stu-id="d7486-128">Design philosophy</span></span>
